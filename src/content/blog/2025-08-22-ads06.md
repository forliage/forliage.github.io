# ads06:回溯(Backtracking)

### **第一部分：回溯算法的理念与基石**

#### **1.1 核心思想**

可以把回溯想象成一种充满智慧的暴力搜索。当我们面对一个问题，可能的解决方案多如牛毛时，纯粹的暴力枚举会让我们陷入“组合爆炸”的困境。而回溯算法，就像一位经验丰富的探险家，它会系统地探索所有可能的路径，但一旦发现某条路是死胡同，它会立刻“回溯”到上一个岔路口，尝试其他选择，而不是一条道走到黑。这种“走不通就退回”的策略，正是回溯算法的精髓。

让我们从一个最基本的问题解决思路开始。对于很多问题，一个“万无一失”的方法是：

1.  **生成所有可能的候选解**。
2.  **逐一检查每个候选解**是否满足问题要求。
3.  最终，从满足要求的解中找出我们需要的答案（比如第一个解、所有解或最优解）。

这个方法虽然可靠，但效率极低。例如，在$8\\times 8$的棋盘上放置8个棋子，有 $C(64, 8)$ 种方式，这是一个天文数字，我们不可能去一一检查。

**回溯算法的出现，就是为了解决这个问题。**

它的核心思想是：**在构建候选解的过程中，实时进行判断，一旦发现当前的“部分解”已经不可能通向一个合法的“完整解”，就立刻停止向这个方向继续探索，并退回到上一步。**

这个提前终止无效探索的过程，我们称之为 **剪枝（Pruning）**。

**回溯算法的本质**

Backtracking enables us to **eliminate** the explicit examination of a large subset of the candidate solutions, while still guaranteeing that the answer will be found if one exists.

简单来说，我们通过剪枝，避免了对整个解空间进行详尽无遗的搜索，从而大大提高了效率。

#### **1.2 状态空间树（State-Space Tree）**

为了更好地理解回溯，我们需要引入一个概念：**状态空间树**。这是一个抽象的树形结构，它代表了问题求解的整个搜索过程：

*   **根节点**：代表搜索的初始状态。
*   **树中的节点**：代表一个“部分解”或中间状态。
*   **从父节点到子节点的边**：代表扩展部分解的一步操作。
*   **叶子节点**：代表一个完整的“候选解”（可能是合法的，也可能不是）。

回溯算法的执行过程，本质上就是对这个状态空间树进行 **深度优先搜索（DFS）**。

在这个图中，当我们从 `C` 走到 `C1` 后，发现这条路径无法产生有效解（比如违反了某个约束条件），我们就会“剪掉”`C1` 下的所有子树，直接返回到 `C`，然后尝试 `C2` 这条路径。这就是剪枝和回溯。

#### **2.2 问题建模与约束分析**

如何用数据结构来表示棋盘和皇后的位置呢？一个非常聪明的做法是使用一个一维数组 $x$。

我们约定 $x\[i\]$ 表示第 $i$ 行的皇后所在的 **列号**。例如 $x\[1\] = 4$ 表示第1行的皇后放在第4列。

这个表示方法本身就隐含了一个强大的约束：**任何两个皇后必定不在同一行**。因为数组的索引 $i$ 就代表了行号，从1到8各不相同。

现在，我们来看剩下的约束条件：

1.  **列约束**: 任何两个皇后不能在同一列。
    
    *   数学表达: $x\[i\] \\neq x\[j\]$ for $i \\neq j$。
    *   这意味着 $x$ 数组中的元素 $x\[1\], x\[2\], ..., x\[8\]$ 必须是 $1, 2, ..., 8$ 的一个 **排列 (Permutation)**。
    *   这个观察非常重要！它将我们的搜索空间从 $8^8$（每行有8种选择）急剧缩小到了 $8!$ (8的阶乘, 40320)，这是一个巨大的进步。
2.  **对角线约束**: 任何两个皇后不能在同一对角线。
    
    *   **数学推理与证明**:
        *   考虑两个皇后，分别位于 $(i, x\[i\])$ 和 $(j, x\[j\])$。
        *   它们在同一 **主对角线** 上，意味着它们的行号与列号之差相等，即 $i - x\[i\] = j - x\[j\]$，可以变形为 $x\[i\] - x\[j\] = i - j$。
        *   它们在同一 **反对角线** 上，意味着它们的行号与列号之和相等，即 $i + x\[i\] = j + x\[j\]$，可以变形为 $x\[i\] - x\[j\] = -(i - j)$。
        *   将这两种情况合并，我们得到 $|x\[i\] - x\[j\]| = |i - j|$。
        *   这等价于 $(x\[i\] - x\[j\]) / (i - j) = ±1$，即两点连线的斜率绝对值为1。
        *   所以，我们的对角线约束就是：对于任意 $i ≠ j$，必须满足 $|x\[i\] - x\[j\]| ≠ |i - j|$。

**总结一下N皇后问题的约束条件:**

*   $x\[i\] = j$ 表示第$i$行第$j$列放一个皇后。
*   **列约束**: $x$ 数组的元素互不相同。
*   **对角线约束**: $|x\[i\] - x\[j\]| ≠ |i - j|$ for all $i ≠ j$。

**Step 2: 带剪枝的深度优先搜索**

现在我们模拟算法的运行过程，假设我们从 $x1 = 1$ 开始。

1.  **放置 Q1**: $x\[1\] = 1$。位置 $(1, 1)$。
2.  **放置 Q2**:
    *   尝试 $x\[2\] = 1$? **失败** (同列)。
    *   尝试 $x\[2\] = 2$? **失败** (对角线)。
    *   尝试 $x\[2\] = 3$? **成功**。当前部分解 $(1, 3)$。
3.  **放置 Q3**:
    *   尝试 $x\[3\] = 2$? **失败** (对角线，与Q2冲突)。
    *   尝试 $x\[3\] = 4$? **失败** (对角线，与Q2冲突)。
    *   所有 $x\[3\]$ 的选择都失败了。这意味着部分解 $(1, 3)$ 无法扩展。
4.  **回溯**:
    *   撤销对 $x\[3\]$ 的放置。
    *   返回到上一步，撤销 $x\[2\] = 3$ 的选择。
5.  **继续放置 Q2**:
    *   尝试 $x\[2\] = 4$? **成功**。当前部分解 $(1, 4)$。
6.  **放置 Q3**:
    *   尝试 $x\[3\] = 2$? **成功**。当前部分解 $(1, 4, 2)$。
7.  **放置 Q4**:
    *   尝试 $x\[3\] = 3$? **失败** (对角线)。
    *   所有 $x\[4\]$ 的选择都失败。
8.  **再次回溯**...

#### **2.4 N皇后问题伪代码与C++实现**

**伪代码:**

```
function NQueens(N):
  // x[i] stores the column for the queen in row i
  let x be an array of size N+1
  
  function backtrack(row):
    if row > N:
      // A solution is found
      printSolution(x)
      return
    
    for col from 1 to N:
      x[row] = col
      if isValid(row, col):
        // This placement is promising, proceed to the next row
        backtrack(row + 1)
      // If isValid is false, or if backtrack(row+1) returns,
      // the loop automatically tries the next column (implicit backtracking).

  function isValid(row, col):
    for i from 1 to row-1:
      // Check column conflict
      if x[i] == col: return false
      // Check diagonal conflict
      if abs(x[i] - col) == abs(i - row): return false
    return true

  // Start the search from the first row
  backtrack(1)
```

**C++代码实现:**

```cpp
#include <iostream>
#include <vector>
#include <cmath>

class NQueensSolution {
private:
    std::vector<std::vector<std::string>> solutions;
    std::vector<int> queens; // queens[i] = column of queen in row i
    int n;

    // Check if placing a queen at (row, col) is valid
    bool isValid(int row, int col) {
        for (int i = 0; i < row; ++i) {
            // Check column conflict
            if (queens[i] == col) {
                return false;
            }
            // Check diagonal conflict
            if (std::abs(row - i) == std::abs(col - queens[i])) {
                return false;
            }
        }
        return true;
    }

    void backtrack(int row) {
        // Base case: If all queens are placed, we found a solution
        if (row == n) {
            std::vector<std::string> board(n, std::string(n, '.'));
            for (int i = 0; i < n; ++i) {
                board[i][queens[i]] = 'Q';
            }
            solutions.push_back(board);
            return;
        }

        // Try placing a queen in each column of the current row
        for (int col = 0; col < n; ++col) {
            if (isValid(row, col)) {
                queens[row] = col; // Place the queen
                backtrack(row + 1); // Recurse to the next row
                // Backtrack is implicit: queens[row] will be overwritten in the next iteration
            }
        }
    }

public:
    std::vector<std::vector<std::string>> solveNQueens(int size) {
        n = size;
        queens.assign(n, -1);
        solutions.clear();
        backtrack(0);
        return solutions;
    }
};

int main() {
    int n = 8;
    NQueensSolution solver;
    std::vector<std::vector<std::string>> result = solver.solveNQueens(n);

    std::cout << "Found " << result.size() << " solutions for " << n << "-Queens problem." << std::endl;
    std::cout << "One of the solutions is:" << std::endl;
    for (const auto& row_str : result[0]) {
        std::cout << row_str << std::endl;
    }
    return 0;
}
```

### **第三部分：回溯应用 - Turnpike Reconstruction Problem**

这是一个更复杂、更能体现回溯中选择与剪枝策略的问题。

#### **3.1 问题描述**

给定 $N$ 个在x轴上的点，其坐标 $x₁ < x₂ < ... < xₙ$。我们不知道这些点的具体坐标，但我们知道所有 $N(N-1)/2$ 个点对之间的距离，这些距离组成一个多重集（multiset） $D$。

**任务**: 根据距离集 $D$，重建出这 $N$ 个点的坐标。 **假设**: $x₁ = 0$。

**示例**: 给定距离集 $D = {1, 2, 2, 2, 3, 3, 3, 4, 5, 5, 5, 6, 7, 8, 10}$。

**Step 1: 确定点的数量 $N$** 距离的数量为 $|D| = 15$。解方程 $N(N-1)/2 = 15$，我们得到 $N² - N - 30 = 0$，即 $(N-6)(N+5)=0$。因为 $N$ 必须是正数，所以 $N=6$。

**Step 2: 确定 $x₁$ 和 $xₙ$**

*   我们假设 $x₁ = 0$。
*   距离集 $D$ 中的最大值一定是 $xₙ$ 和 $x₁$ 之间的距离。
*   所以 $xₙ = \\max(D) = 10$。
*   现在我们已经确定了 $x₁ = 0$ 和 $x₆ = 10$。我们需要找到 $x₂, x₃, x₄, x₅$。

**Step 3: 回溯搜索策略** 这是一个关键点。我们应该如何选择下一个要确定的点？一个非常有效的启发式策略是：**总是尝试用 $D$ 中当前最大的未使用的距离来放置下一个点**。

设 $d\_\\text{max}$ 是 $D$ 中当前最大的距离。这个距离 $d\_\\text{max}$ 必定是 $xⱼ - xᵢ$ 的结果。由于我们已经确定了最左 $x₁$ 和最右 $x₆$，下一个最大的距离很可能是由 $x₁$ 或 $x₆$ 产生的。

这就给了我们两个主要的猜测（或选择）：

1.  **选择 1**: 存在一个点 $xₖ$，使得 $xₖ - x₁ = d\_\\text{max}$，即 $xₖ = d\_\\text{max}$。
2.  **选择 2**: 存在一个点 $xₖ$，使得 $x₆ - xₖ = d\_\\text{max}$，即 $xₖ = x₆ - d\_\\text{max}$。

我们的回溯算法将围绕这两个选择展开：

1.  从 $D$ 中取出最大距离 $d\_\\text{max}$。
2.  **尝试选择 1**:
    *   假设要放置的点坐标是 $y = d\_\\text{max}$。
    *   **剪枝检查 (Check)**: 检查 $y$ 与所有已确定点 $xᵢ$ 的距离 $|y - xᵢ|$ 是否都存在于当前的距离集 $D$ 中。
    *   如果检查通过，就从 $D$ 中删除这些距离，然后递归地解决子问题。
    *   如果递归失败了，**回溯**：将刚才删除的距离加回到 $D$ 中。
3.  **尝试选择 2** (如果选择1失败或我们需要找到所有解):
    *   假设要放置的点坐标是 $y = xₙ - d\_\\text{max}$。
    *   重复上述的剪枝检查、递归和回溯过程。
4.  如果两个选择都失败了，说明之前的某个步骤做出了错误的选择，函数返回 `false`，触发更深层次的回溯。

**示例 walkthrough**:

*   $N=6$, $x₁=0, x₆=10$。$D = {1, 2, 2, 2, 3, 3, 3, 4, 5, 5, 5, 6, 7, 8}$ (10已被使用)。
*   $d\_\\text{max} = 8$。
    *   **选择1**: 尝试 $x₅ = 8$。
        *   **Check**:
            *   $|x₅ - x₁| = |8 - 0| = 8$ (存在于$D$)。
            *   $|x₅ - x₆| = |8 - 10| = 2$ (存在于$D$)。
        *   检查通过！从 $D$ 中移除 $8$ 和 $2$。递归。
*   新 $D$ 的 $d\_\\text{max} = 7$。
    *   **选择1**: 尝试 $x₄ = 7$。
        *   **Check**:
            *   $|x₄ - x₁| = |7 - 0| = 7$ (存在)。
            *   $|x₄ - x₆| = |7 - 10| = 3$ (存在)。
            *   $|x₄ - x₅| = |7 - 8| = 1$ (存在)。
        *   检查通过！从 $D$ 中移除 $7$, $3$, $1$。递归。
*   ...这个过程继续下去。如果某一步走不通（例如，某个计算出的距离在 $D$ 中找不到），算法就会回溯，尝试另一个选择（例如，尝试 $x₄ = x₆ - 7 = 3$）。

最终，算法会找到正确的解： **(0, 3, 5, 6, 8, 10)**。 我们可以验证这个解生成的距离集是否与 $D$ 匹配。

#### **3.2 C++代码框架**

```
// DistSet D could be a multiset or a frequency map for efficient operations
bool Reconstruct(DistType X[], DistSet& D, int N, int left, int right) {
    // Base Case: all points are placed
    if (D.is_empty()) {
        return true; 
    }

    // Heuristic: Try to place a point using the largest remaining distance
    DistType d_max = D.find_max();

    // --- Option 1: Place point at d_max ---
    bool ok_option1 = check_distances(d_max, X, left, right, D);
    if (ok_option1) {
        X[right-1] = d_max; // Place the new point
        // Remove distances from D
        remove_distances(d_max, X, left, right, D);

        if (Reconstruct(X, D, N, left, right - 1)) {
            return true; // Solution found
        }
        // Backtrack: if it didn't work, undo the changes
        add_distances(d_max, X, left, right, D);
    }
    
    // --- Option 2: Place point at X[N] - d_max ---
    DistType y = X[N] - d_max;
    bool ok_option2 = check_distances(y, X, left, right, D);
    if (ok_option2) {
        X[left] = y; // Place the new point
        // Remove distances from D
        remove_distances(y, X, left, right, D);

        if (Reconstruct(X, D, N, left + 1, right)) {
            return true;
        }
        // Backtrack
        add_distances(y, X, left, right, D);
    }

    // Both options failed
    return false;
}
```

### **第四部分：回溯算法的通用模板与优化**

#### **4.1 通用模板**

几乎所有的回溯问题都可以套用下面这个模板：

```
bool Backtracking(int i, State& currentState) {
    // Base case: Have we built a complete solution?
    if (i > N) {
        // Yes, we have a valid solution (x1, ..., xN)
        return true; 
    }

    // Loop through all possible choices for the i-th element
    for (each choice x_i in CandidateSet S_i) {
        
        // Check if this choice is valid given the current state
        if (Check(x_i, currentState)) {
            
            // Apply the choice
            Count(x_i, currentState); 

            // Recurse to the next level
            if (Backtracking(i + 1, currentState)) {
                return true; // Found a solution, propagate success up
            }

            // Backtrack: Undo the choice
            Undo(x_i, currentState); 
        }
    }

    // No valid choice was found at this level
    return false;
}
```

*   `i`: 当前正在处理的决策层级。
*   `CandidateSet S_i`: 第 `i` 步的所有可能选择。
*   `Check`: 剪枝函数，判断当前选择是否合法。
*   `Count` / `Undo`: 更新和恢复状态的操作。

#### **4.2 优化：优先探索更小的搜索空间**

当不同决策步骤的候选集大小 (`S_i`) 不同时，有一个重要的启发式优化：**优先处理选择最少的步骤 (Most Constrained Variable)**。

为什么呢？因为选择越少，越容易碰到死胡同。尽早碰到死胡同，就能尽早剪枝，避免在注定失败的分支上浪费大量时间。

### **第五部分：拓展 - 游戏博弈与Alpha-Beta剪枝**

回溯是许多AI游戏算法的基础，比如让AlphaGo成名的蒙特卡洛树搜索，其底层也离不开树的遍历思想。我们来看一个更简单的例子：井字棋（Tic-tac-toe）。

#### **5.1 Minimax策略**

在两人零和博弈中（一方的收益就是另一方的损失），我们可以使用**Minimax算法**来寻找最优走法。

*   **思想**: 假设对手每一步都会选择对我们最不利的走法，而我们则在所有可能的走法中，选择那个在最坏情况下结果最好的走法。
*   **实现**:
    *   构建游戏状态树。
    *   定义一个**评估函数 `f(P)`** 来量化一个局面 `P` 的“好坏程度”。例如：`f(P) = (我方能赢的路线数) - (对方能赢的路线数)`。
    *   我方（**MAX**层）的目标是最大化 `f(P)`。
    *   对方（**MIN**层）的目标是最小化 `f(P)`。
    *   从叶子节点（游戏结束或达到搜索深度）开始，向上层层传递评价值：MIN层选择子节点中的最小值，MAX层选择子节点中的最大值。

#### **5.2 Alpha-Beta剪枝**

Minimax算法需要遍历整棵游戏树，效率不高。**Alpha-Beta剪枝**是其革命性的优化，它属于回溯思想在博弈树中的应用。

*   **核心思想**: 在搜索过程中，如果发现某个分支再怎么搜索，其最终的评估值也肯定不会比当前已经找到的其他选择更好，那么这个分支就可以被安全地“剪掉”。
*   **两个关键值**:
    *   **Alpha (α)**: 在MAX玩家的路径上，目前已经找到的**最好**（最大）的分数。MAX玩家的目标是让最终分数 ≥ α。
    *   **Beta (β)**: 在MIN玩家的路径上，目前已经找到的**最差**（最小）的分数。MIN玩家的目标是让最终分数 ≤ β。
*   **剪枝条件**: 当 `α ≥ β` 时，可以进行剪枝。

**1\. α 剪枝 (发生在MIN节点)**

*   **场景**: MAX玩家正在评估一个走法，进入了一个MIN节点。
*   **逻辑**: 如果这个MIN节点下的某个分支返回了一个值 `v`，而这个 `v` 比MAX玩家在**其他分支**已经确保能得到的最低分 `α` 还要差（即 `v ≤ α`），那么MAX玩家绝不会选择这条路。因为对手（MIN）只会让情况变得更糟（得到比 `v` 更小或等于 `v` 的值）。所以，这个MIN节点下的其他分支就没必要再看了。

*   **场景**: MIN玩家正在评估对手的一个走法，进入了一个MAX节点。

**效果**: Alpha-Beta剪枝在理想情况下，可以将搜索的节点数从 `O(b^d)` 降低到 `O(b^(d/2))`，其中 `b` 是分支因子，`d` 是搜索深度。这相当于用同样的时间，可以将搜索深度加倍，棋力呈指数级提升。
