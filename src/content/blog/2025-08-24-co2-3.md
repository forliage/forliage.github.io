# 计算机组成2-3:逻辑运算与决策指令

### **6\. 逻辑运算**

到目前为止，我们接触的运算主要是算术运算（加、减）。但计算机的能力远不止于此。另一大类至关重要的运算是**逻辑运算**。在高级语言中，我们可能只在布尔表达式（如 `if (a && b)`）中见过它，但在计算机底层，逻辑运算的核心作用是**按位操作 (Bitwise Manipulation)**。

这意味着我们可以对一个64位寄存器中的每一位独立进行操作，像是有64个并排的小开关一样。这对于提取、插入或修改一个数据字中的特定位域（groups of bits）非常有用。

#### **6.2 按位与 (AND)**

`and rd, rs1, rs2` 或 `andi rd, rs1, immediate`

**规则**：只有当两个操作数的对应位都为1时，结果的对应位才为1；否则为0。

**核心应用：掩码 (Masking) —— 清零特定位** AND运算就像一个“过滤器”或“遮罩”。我们可以构造一个特定的“掩码”数，其中我们**关心**的位设为1，**不关心**（希望清零）的位设为0。将原始数据与这个掩码进行AND运算，就能精确地保留我们想要的位，同时将其他位全部清零。

**示例**：假设寄存器 `x10` 的值为 `...00001101 11000000`，我们只想保留其低8位中的高4位（即`1101`中的`1100`部分），将其他位全部清零。

1.  **构造掩码**：我们需要的掩码是 `...00000000 00110000`。这个掩码被存放在 `x11` 中。
2.  **执行AND**：`and x9, x10, x11`

```
  x10:  ... 00000000 00000000 00001101 11000000
                & x11:  ... 00000000 00000000 00000000 00110000  (掩码)
                --------------------------------------------------
                = x9:   ... 00000000 00000000 00000000 00000000
                
```

_糟糕，我这里犯了个错误，让我们用一个更好的例子，对应PDF中的例子_ **示例（修正）**：从 `x10` 中提取 `00001101` 部分。

1.  **构造掩码**：`x11` 中存放掩码 `...00000000 00111100`
2.  **执行AND**：`and x9, x10, x11`

```
  x10:  ... 00001101 11000000
                & x11:  ... 00111100 00000000
                -----------------------------
                = x9:   ... 00001100 00000000
                
```

结果 `x9` 中精确地保留了我们感兴趣的位，其余位都被“屏蔽”掉了。

#### **6.3 按位或 (OR)**

`or rd, rs1, rs2` 或 `ori rd, rs1, immediate`

**规则**：只要两个操作数的对应位中**至少有一个**为1，结果的对应位就为1。

**核心应用：置位 (Setting) —— 将特定位置为1** OR运算可以强制将数据的某些位置为1，而不影响其他位。我们构造一个掩码，其中需要**置为1**的位设为1，需要**保持不变**的位设为0。

**示例**：我们希望将寄存器 `x10` 的值 `...00001101 11000000` 的低16位中的 `...00111100 00000000` 这部分强制设为1，而不改变其他位。

1.  **构造掩码**：`x11` 中存放掩码 `...00000000 00111100 00000000`
2.  **执行OR**：`or x9, x10, x11`

```
  x10:  ... 00001101 11000000
                | x11:  ... 00111100 00000000
                -----------------------------
                = x9:   ... 00111101 11000000
                
```

结果 `x9` 中，掩码为1的位都被强制置为1了，而掩码为0的位则保持了 `x10` 的原始值。

#### **6.4 按位异或 (XOR)**

`xor rd, rs1, rs2` 或 `xori rd, rs1, immediate`

**规则**：当两个操作数的对应位**不相同**时，结果的对应位为1；相同时为0。

**核心应用：翻转/取反 (Flipping/Inverting) —— 将特定位反转** XOR运算有一个神奇的特性：`x ^ 0 = x` (与0异或，保持不变)，`x ^ 1 = ~x` (与1异或，等于取反)。这使得XOR成为一个可编程的“反相器”。

**示例**：实现 `NOT` 操作。虽然RISC-V没有单独的 `not` 指令，但我们可以通过 `xori` 实现。要对寄存器 `x10` 的所有位取反，只需将其与全1的数（即-1的补码）进行异或。

1.  **构造掩码**：`x12` 中存放 `-1`，其二进制表示为全1: `1111...1111`
2.  **执行XOR**：`xor x9, x10, x12`

```
  x10:  ... 00001101 11000000
                ^ x12:  ... 11111111 11111111
                -----------------------------
                = x9:   ... 11110010 00111111
                
```

可以看到，`x9` 的每一位都与 `x10` 的对应位相反。

**思考：** 为什么RISC-V不提供`not`指令？这再次体现了RISC的设计哲学。`not`操作可以被`xori`完美替代，单独为其设计一条指令会增加硬件的复杂性，而收益不大。保持指令集的精简是首要目标。

#### **6.5 移位运算 (Shift Operations)**

移位运算移动寄存器中各位的位置。它们在算术上等同于乘以或除以2的幂，但在硬件上比真正的乘除法器快得多，是性能优化的关键。

*   **逻辑左移 (Shift Left Logical: `sll`, `slli`)** `slli rd, rs1, shamt` (shamt: shift amount, 移位量) 将 `rs1` 中的各位向左移动 `shamt` 位，右边空出的位用**0**填充。 **算术意义**：相当于无符号数乘以 $2^{\\text{shamt}}$。 例如，`slli x5, x5, 3` 相当于 `x5 = x5 * 8`。
    
*   **逻辑右移 (Shift Right Logical: `srl`, `srli`)** 将 `rs1` 中的各位向右移动 `shamt` 位，左边空出的位用**0**填充。 **算术意义**：相当于**无符号数**除以 $2^{\\text{shamt}}$。
    
*   **算术右移 (Shift Right Arithmetic: `sra`, `srai`)** 将 `rs1` 中的各位向右移动 `shamt` 位，左边空出的位用**原始符号位的副本**填充。 **为什么需要它？** 为了正确处理负数。如果一个负数（符号位为1）逻辑右移，左边补0，符号位会变成0，数就从负数变成了正数，这在算术上是错误的。算术右移通过复制符号位，保证了负数在右移后仍然是负数。 **算术意义**：相当于**有符号数**除以 $2^{\\text{shamt}}$。
    

### **7\. 决策指令**

如果程序只能从头到尾顺序执行，那么计算机只不过是一个高级计算器。软件的强大威力在于其能够根据不同的条件执行不同的代码路径，这就是**控制流 (Control Flow)**。`if-else`、`while`、`for`、`switch` 等结构都是控制流的体现。

硬件上，控制流是通过**分支 (Branch)** 和 **跳转 (Jump)** 指令实现的。

#### **7.1 条件分支：`if` 语句的基石**

RISC-V提供了多条条件分支指令，它们根据两个寄存器的比较结果来决定是否改变程序的执行顺序。

*   `beq rs1, rs2, Label` (Branch on Equal): 如果 `rs1 == rs2`，则跳转到 `Label` 标记的指令处执行。
*   `bne rs1, rs2, Label` (Branch on Not Equal): 如果 `rs1 != rs2`，则跳转到 `Label` 标记的指令处执行。

**示例1：编译 `if (i == j) f = g + h;`** 假设 `f,g,h,i,j` 分别在 `x19, x20, x21, x22, x23`。

**逻辑**：如果 `i` 和 `j` **不相等**，就跳过 `add` 指令。

```
    # if (i != j) goto Next;
                    bne  x22, x23, Next
                    # f = g + h;
                    add  x19, x20, x21
                Next:
                    # ... 后续代码
                
```

**示例2：编译 `if (i == j) f = g + h; else f = g - h;`**

**逻辑**：如果 `i` 和 `j` **不相等**，就跳转到 `else` 部分的代码。如果相等，则顺序执行 `then` 部分，并在其后用一条**无条件跳转**跳过 `else` 部分。

```
    # if (i != j) goto Else;
                    bne  x22, x23, Else
                    # "then" part: f = g + h;
                    add  x19, x20, x21
                    # unconditionally skip the Else part
                    beq  x0, x0, Exit   # Tip: beq x0, x0 is an unconditional branch!
                Else:
                    # "else" part: f = g - h;
                    sub  x19, x20, x21
                Exit:
                    # ... 后续代码
                
```

#### **7.2 用分支构建循环**

循环的本质就是在一个代码块的末尾，有一个条件分支决定是跳回代码块的开头继续执行，还是跳出循环。

**示例：编译 `while (save[i] == k) i = i + 1;`** 假设 `save` 基地址在 `x25`，`i` 在 `x22`，`k` 在 `x24`。

```
Loop:
                    # 计算 save[i] 的地址
                    slli x10, x22, 3        # temp x10 = i * 8
                    add  x10, x10, x25      # x10 = address of save[i]
                    # 从内存加载 save[i]
                    ld   x9, 0(x10)         # temp x9 = save[i]
                    # 比较并分支: if (save[i] != k) goto Exit
                    bne  x9, x24, Exit
                    # 循环体: i = i + 1
                    addi x22, x22, 1
                    # 无条件跳回循环开始处
                    beq  x0, x0, Loop
                Exit:
                    # ... 循环结束后的代码
                
```

#### **7.3 更多比较：小于、大于**

`beq` 和 `bne` 只能处理相等性判断。如何实现 `<`, `>`, `<=`, `>=` 呢？RISC-V提供了两种方式：

**方式一：先比较，后分支**

使用 `slt` (Set on Less Than) 指令系列。

*   `slt rd, rs1, rs2`: 如果 `rs1 < rs2` (有符号比较)，则 `rd = 1`，否则 `rd = 0`。
*   `sltu rd, rs1, rs2`: 功能同上，但进行的是**无符号比较**。
*   `slti`, `sltiu` 是立即数版本。

**示例：编译 `if (a < b) goto Less;`** 假设 `a` 在 `x8`，`b` 在 `x9`。

```
    slt  x5, x8, x9     # x5 = (a < b) ? 1 : 0
                    # if (x5 != 0) goto Less
                    bne  x5, x0, Less
                
```

通过 `slt` 和 `bne` 的组合，我们可以实现“小于则跳转”。其他比较（`>`、`>=`、`<=`）也可以通过颠倒操作数或判断 `slt` 结果是否为0来实现。

**方式二：一步到位**

RISC-V也提供了直接进行大小比较并分支的指令，这更高效：

*   `blt rs1, rs2, Label` (Branch on Less Than, signed)
*   `bge rs1, rs2, Label` (Branch on Greater Than or Equal, signed)
*   `bltu rs1, rs2, Label` (Branch on Less Than, unsigned)
*   `bgeu rs1, rs2, Label` (Branch on Greater Than or Equal, unsigned)

**示例：编译 `if (a > b) a = a + 1;`** 假设 `a` 在 `x22`，`b` 在 `x23`。 逻辑：如果 `b >= a`，则跳过加法。

```
    bge  x23, x22, Exit    # Branch if b >= a
                    addi x22, x22, 1      # a = a + 1
                Exit:
                    # ...
                
```

**有符号 vs. 无符号：一个致命的陷阱！** 选择错误的比较指令是C语言中一类非常隐蔽且危险的bug的根源。 考虑两个32位寄存器：

*   `x22 = 1111 ... 1111`
    
*   `x23 = 0000 ... 0001`
    
*   **有符号比较 (`blt`)**: 编译器会解释 `x22` 为 `-1`，`x23` 为 `+1`。因为 `-1 < +1`，所以 `blt x22, x23, ...` 会发生跳转。
    
*   **无符号比较 (`bltu`)**: 编译器会解释 `x22` 为 $2^{32}-1$（一个巨大的正数），`x23` 为 `+1`。因为 $2^{32}-1 > +1$，所以 `bltu x22, x23, ...` **不会**发生跳转。
    

**结论**：对同样的一串0和1，采用不同的解释（约定），会得到截然相反的比较结果！

#### **7.4 `switch` 语句：高效的多路分支**

如果用一长串的 `if-else-if` 来实现 `switch` 语句，当 `case` 很多时，程序需要进行多次比较和分支，效率低下。

**优化方案：跳转地址表 (Jump Address Table)** 这是一种利用内存和间接跳转来实现高效多路分支的经典技巧。

**原理：**

1.  **创建表**：编译器在内存中创建一个表（数组），表的每个元素都是一个地址，对应一个 `case` 代码块的入口。
2.  **索引**：将 `switch` 的变量 `k` 作为索引。
3.  **计算地址**：计算 `k` 对应的表项在内存中的地址 ( `表基地址 + k * 8`，因为每个地址是64位的，占8字节)。
4.  **加载目标**：从内存中加载这个地址到寄存器中。
5.  **间接跳转**：使用 `jalr` (Jump and Link Register) 指令，跳转到该寄存器中保存的地址。

**RISC-V 代码片段 (示意)**:

```
    # 假设 switch 变量 k 在 x25, 跳转表基地址在 x6
                    # 边界检查 (e.g., if k < 0 or k > 3, goto default_case)
                    blt  x25, x0, default_case
                    addi x5, x0, 3          # 假设有 4 个 case (0, 1, 2, 3)
                    bge  x25, x5, default_case

                    # 计算表项地址
                    slli x7, x25, 3         # x7 = k * 8
                    add  x7, x7, x6          # x7 = &JumpTable[k]

                    # 加载目标代码地址
                    ld   x7, 0(x7)           # x7 = JumpTable[k]

                    # 跳转
                    jalr x0, 0(x7)          # Jump to address in x7
                
```

这种方法无论有多少个 `case`，都只需要一次内存访问和一次跳转，效率极高。
