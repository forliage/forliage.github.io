# 计算机动画12:隐函数曲面造型和动画技术

### **第一部分：三维世界的几何表示法**

在进入隐函数的世界之前，我们首先需要了解在计算机图形学中，我们有哪些主流的工具来描述一个三维物体的形状。把这些工具想象成艺术家的不同媒介：有的人用积木，有的人用画笔，有的人用黏土。

*   **参数曲面 (Parametric Surface)**：如Bezier曲面、NURBS曲面。它们通过一个二维参数 $(u, v)$ 映射到三维空间中的点 $(x, y, z)$ 来定义曲面。你可以把它想象成在一块有弹性的橡胶布上（UV空间）画画，然后把这块布拉伸、扭曲成三维形状。这种方法非常适合构建光滑、精确的工业设计表面，比如汽车车身。
    
*   **多边形网格 (Polygon Mesh)**：这是最普遍、最直接的表示方法。它将物体的表面近似为由许多微小的、平坦的多边形（通常是三角形或四边形）拼接而成。我们玩的绝大多数3D游戏中的模型，都是多边形网格。
    
*   **细分曲面 (Subdivision Surface)**：它结合了前两者的优点。它从一个粗糙的多边形控制网格出发，通过一套递归的细分规则，不断地增加顶点并平滑其位置，最终生成一个光滑的曲面。Pixar动画电影中的角色模型大量使用了这项技术。
    

以上三种，无论是参数曲面还是多边形网格，都属于**边界表示 (Boundary Representation, B-Rep)**。它们直接描述了物体的“皮肤”，但对于物体“内部”的信息则知之甚少。它们在处理某些情况时会遇到困难：

*   **拓扑变化困难**：想象一下，要模拟两滴水融合在一起。对于多边形网格来说，你需要极其复杂的算法来检测碰撞、删除重叠的三角面、然后重新“缝合”网格，这个过程非常繁琐且容易出错。
*   **体模型表示不直观**：像克莱因瓶这样没有严格内外之分的曲面，用边界表示法来处理会很奇怪。

而我们今天的主角——**隐式曲面 (Implicit Surface)**，则提供了一种全新的思路。

### **第二部分：隐式曲面——用场来定义形状**

隐式曲面不直接描述“皮”，而是定义了一个遍布整个三维空间的**标量场 (Scalar Field)**。

#### **2.1 核心思想：等值面 (Isosurface)**

想象一下地理课上的等高线地图。地图上的每一条等高线，连接了所有海拔相同的点。例如，海拔800米等高线上的所有点，其`海拔(经度, 纬度) = 800`。

现在，我们将这个思想扩展到三维空间。我们定义一个函数 $f(x, y, z)$，它为空间中任意一个点赋予一个标量值。这个函数就构建了一个标量场。**隐式曲面就是这个场中所有函数值等于某个特定阈值（常数）$T$ 的点的集合。**

在数学上，这个曲面 $S$ 被定义为： $$S = { (x, y, z) \\in \\mathbb{R}^3 \\mid f(x, y, z) = T }$$ 这个曲面 $S$ 就被称为函数 $f$ 的**等值面 (Isosurface)** 或**水平集 (Level Set)**。通常为了方便，我们会将阈值 $T$ 移到函数一侧，并令其为0，即： $$g(x, y, z) = f(x, y, z) - T = 0$$

#### **2.2 内外之分：空间的划分**

隐式曲面的一个巨大优势是，它不仅定义了曲面本身，还自然地将整个空间划分为三个区域：

*   **曲面上 (On)**：满足 $f(x, y, z) = 0$ 的点。
*   **内部 (Inside)**：满足 $f(x, y, z) > 0$ 的点。
*   **外部 (Outside)**：满足 $f(x, y, z) < 0$ 的点。

当然，你也可以反过来定义，让 $f<0$ 为内部。关键在于保持定义的一致性。这个特性使得**点的内外判断 (Inside/Outside Test)** 变得极其简单高效：只需计算该点的函数值并检查其符号即可。这在碰撞检测等应用中非常有用。

```
// C++ 伪代码
                enum PointLocation { INSIDE, OUTSIDE, ON_SURFACE };

                PointLocation check_location(const glm::vec3& point) {
                    float value = implicit_function(point);
                    if (abs(value) < 1e-6) { // 使用一个小容差
                        return ON_SURFACE;
                    } else if (value > 0) {
                        return INSIDE;
                    } else {
                        return OUTSIDE;
                    }
                }
                
```

#### **2.3 曲面法向：梯度的威力**

在图形学中，曲面的法向量至关重要，它决定了光照和着色的效果。对于隐式曲面，法向量的计算异常优雅：**曲面上任意一点的法向量，就是该点处函数的梯度方向。**

**数学证明**： 梯度 $\\nabla f$ 定义为： $$\\nabla f(x, y, z) = \\left( \\frac{\\partial f}{\\partial x}, \\frac{\\partial f}{\\partial y}, \\frac{\\partial f}{\\partial z} \\right)$$ 梯度向量指向函数值增长最快的方向。为什么它垂直于等值面呢？

1.  考虑曲面 $f(x, y, z) = C$ 上的任意一点 $\\mathbf{p}\_0 = (x\_0, y\_0, z\_0)$。
2.  再考虑曲面上的一条经过 $\\mathbf{p}\_0$ 的任意平滑曲线，其参数方程为 $\\mathbf{p}(t) = (x(t), y(t), z(t))$，且 $\\mathbf{p}(0) = \\mathbf{p}\_0$。
3.  因为曲线上的所有点都在等值面上，所以对于任意的 $t$，都有 $f(\\mathbf{p}(t)) = C$。
4.  我们将这个方程对 $t$ 求导。根据多元复合函数求导的链式法则，我们得到： $\\frac{d}{dt} f(\\mathbf{p}(t)) = \\frac{\\partial f}{\\partial x} \\frac{dx}{dt} + \\frac{\\partial f}{\\partial y} \\frac{dy}{dt} + \\frac{\\partial f}{\\partial z} \\frac{dz}{dt} = \\frac{dC}{dt} = 0$
5.  这个表达式可以写成两个向量的点积形式： $$\\left( \\frac{\\partial f}{\\partial x}, \\frac{\\partial f}{\\partial y}, \\frac{\\partial f}{\\partial z} \\right) \\cdot \\left( \\frac{dx}{dt}, \\frac{dy}{dt}, \\frac{dz}{dt} \\right) = 0$$
6.  即 $\\nabla f(\\mathbf{p}(t)) \\cdot \\mathbf{p}'(t) = 0$。
7.  $\\mathbf{p}'(t)$ 正是曲线在 $\\mathbf{p}(t)$ 点的切向量。这个等式意味着，函数在曲面上一点的**梯度向量**，与经过该点的**任意一条曲面切向量**都垂直。
8.  因此，梯度向量 $\\nabla f$ 必然是该点处曲面的**法向量**。

这个性质非常强大。我们无需复杂的几何计算，只需对函数求偏导，就能得到法向量。

*   如果 $f>0$ 表示内部，那么梯度 $\\nabla f$ 指向内部（函数值增大的方向）。
*   如果 $f<0$ 表示内部，那么梯度 $\\nabla f$ 指向外部。

### **第三部分：隐式曲面的“超能力”**

基于以上核心思想，隐式曲面展现出了一些传统几何表示法难以匹敌的强大能力。

#### **3.1 超能力一：轻松的构造性实体几何 (CSG)**

CSG 是一种通过对基本几何体进行布尔运算（并、交、差）来构建复杂模型的技术。对于多边形网格，实现CSG非常复杂。但对于隐式曲面，它出奇地简单！

假设我们有两个隐式曲面，分别由函数 $f(x,y,z)$ 和 $g(x,y,z)$ 定义，并且我们约定 $f<0$ 和 $g<0$ 表示物体内部。

*   **并集 (Union, $A \\cup B$)**: 一个点在并集中，当且仅当它在 A **或** B 中。这等价于 $f < 0$ **或** $g < 0$。这个逻辑可以用 `min` 函数完美实现： $$f\_{\\text{union}}(x,y,z) = \\min(f(x,y,z), g(x,y,z))$$ 因为只有当 $f$ 和 $g$ 中至少一个为负时，它们的最小值才为负。
    
*   **交集 (Intersection, $A \\cap B$)**: 一个点在交集中，当且仅当它同时在 A **和** B 中。这等价于 $f < 0$ **且** $g < 0$。这个逻辑可以用 `max` 函数实现： $$f\_{\\text{intersection}}(x,y,z) = \\max(f(x,y,z), g(x,y,z))$$ 因为只有当 $f$ 和 $g$ 同时为负时，它们的最大值才为负。
    
*   **差集 (Subtraction, $A - B$)**: 一个点在差集中，当且仅当它在 A 中，但不在 B 中。这等价于 $f < 0$ **且** $g \\ge 0$。这可以通过交集和补集实现：$A \\cap (\\text{not } B)$。 一个物体的补集（反转内外）可以通过对函数取反得到。所以，B 的外部由 $-g < 0$ 定义。因此，差集就是 A 与 B 的补集的交集： $$f\_{\\text{subtraction}}(x,y,z) = \\max(f(x,y,z), -g(x,y,z))$$
    

这种用简单的 `min/max` 操作就能实现复杂几何运算的能力，是隐式曲面的一大魅力。

#### **3.2 超能力二：自然的拓扑变化与融合**

还记得T-1000吗？它最酷的能力就是融合。这正是隐式曲面最擅长的领域。这个技术被称为**元球 (Metaball)** 或 **Blobby Model**。

**元球造型 (Metaball Modeling)** 的思想始于上世纪80年代。其核心是：

1.  空间中放置若干个基本形状（元球），每个元球都定义了一个以其中心为最高点、向外平滑衰减的标量场。
2.  在空间中任意一点的总场强，是所有元球在该点场强的**总和**。
3.  最终的曲面就是总场强等于某个阈值的等值面。

当两个元球相互靠近时，它们的场会叠加。在它们之间区域的总场强会升高，当这个场强超过阈值时，两个独立的表面就会自然地、平滑地**融合 (Blend)** 在一起，形成一个连通的表面。这个过程不需要任何复杂的网格操作，拓扑变化是自动发生的。

#### **3.3 超能力三：高效的求交计算**

在光线追踪渲染中，我们需要计算光线与物体的交点。对于隐式曲面，这个过程也很直接。 一条光线的参数方程可以表示为： $$\\mathbf{P}(t) = \\mathbf{O} + t \\cdot \\mathbf{D}$$ 其中 $\\mathbf{O}$ 是光线起点，$\\mathbf{D}$ 是方向向量，$t$ 是距离。

要找到交点，我们只需将光线方程代入隐式曲面方程 $f(\\mathbf{P}) = 0$，即： $$f(\\mathbf{O} + t \\cdot \\mathbf{D}) = 0$$ 这会得到一个只含未知数 $t$ 的方程。我们只需解出最小的正实数根 $t$，就能找到第一个交点。

**示例：光线与椭球求交** 椭球的隐式方程为： $$(\\frac{x}{r\_x})^2 + (\\frac{y}{r\_y})^2 + (\\frac{z}{r\_z})^2 - 1 = 0$$ 代入光线方程 $\\mathbf{P}(t) = (O\_x+tD\_x, O\_y+tD\_y, O\_z+tD\_z)$： $$(\\frac{O\_x+tD\_x}{r\_x})^2 + (\\frac{O\_y+tD\_y}{r\_y})^2 + (\\frac{O\_z+tD\_z}{r\_z})^2 - 1 = 0$$ 这是一个关于 $t$ 的一元二次方程 $At^2 + Bt + C = 0$，可以用求根公式轻松解出。对于更高次的代数曲面（如心形曲面），虽然解析解可能不存在，但我们仍然可以使用牛顿法等数值方法来求解。

### **第四部分：构建隐式曲面的方法**

我们已经知道了隐式曲面是什么以及它的优点，那么，我们具体如何定义那个神奇的函数 $f(x, y, z)$ 呢？这里有几种主流的方法。

#### **4.1 代数曲面 (Algebraic Surfaces)**

这是最直接的方法，即用一个多项式方程来定义曲面。 **二次代数曲面 (Quadrics)** 是最常见的，其通用方程为： $$f(x,y,z) = Ax^2 + Ey^2 + Hz^2 + 2Bxy + 2Cxz + 2Fyz + 2Dx + 2Gy + 2Iz + J = 0$$ 通过选择不同的系数，这个方程可以表示球、椭球、圆柱、圆锥、双曲面、抛物面等所有二次曲面。

我们还可以使用**高次代数曲面**来创建更复杂的形状，比如心形、四叶结等。这些形状虽然优美，但通过调整方程系数来直观地控制形状非常困难。

#### **4.2 距离曲面 (Distance Surfaces) 与距离场 (Distance Field)**

这是一种非常强大和直观的方法。函数 $f(x, y, z)$ 的值被定义为点 $(x, y, z)$ 到某个几何对象（如点、线、曲线、甚至一个复杂模型）的**最短距离**。 $$f(\\mathbf{x}, \\text{Object}) = d(\\mathbf{x}, \\text{Object}) - r = \\min\_{\\mathbf{y} \\in \\text{Object}} ||\\mathbf{x} - \\mathbf{y}|| - r$$ 当 $r=0$ 时，等值面就是物体本身。当 $r>0$ 时，就是物体向外膨胀 $r$ 距离后的表面。

*   **球**: $f(\\mathbf{x}) = d(\\mathbf{x}, \\text{point}) - r = ||\\mathbf{x} - \\mathbf{c}|| - r = 0$
*   **圆柱**: $f(\\mathbf{x}) = d(\\mathbf{x}, \\text{line}) - r = 0$
*   **圆环**: $f(\\mathbf{x}) = d(\\mathbf{x}, \\text{circle}) - r = 0$

当这个距离函数被定义为**有向距离函数 (Signed Distance Function, SDF)** 时，即在物体内部为负，外部为正，我们就得到了一个**距离场 (Distance Field)**。这是一个极其有用的数据结构，广泛应用于实时渲染、碰撞检测和三维重建（如Kinect Fusion, Poisson Reconstruction）中。

#### **4.3 卷积曲面 (Convolution Surfaces)**

距离曲面在处理由线段构成的骨架时，会在连接处产生不平滑的“鼓包”(bulge)。卷积曲面正是为了解决这个问题而提出的。 其核心思想是通过**卷积**运算来生成标量场。 $$f(\\mathbf{P}) = (g \* h)(\\mathbf{P}) = \\int\_V g(\\mathbf{Q}) h(\\mathbf{P} - \\mathbf{Q}) dV$$

*   $g(\\mathbf{Q})$: **几何函数**，定义了物体的“骨架”(skeleton)。在骨架上 $g=1$，其他地方 $g=0$。
*   $h(\\mathbf{r})$: **核函数 (Kernel Function)**，通常是一个径向对称的、平滑衰减的函数，如高斯函数 $h(r) = e^{-a^2 r^2}$。它扮演着一个低通滤波器的角色。

直观地理解，卷积过程就是将核函数沿着骨架进行“涂抹”或“模糊化”。骨架上的每一点都向外辐射一个场，空间中任意一点的场强是所有这些辐射的累加。这保证了即使在骨架的尖锐连接处，生成的场也是平滑的。

#### **4.4 元球/Blobby模型 (离散化的卷积)**

元球可以看作是卷积曲面的一种离散化和简化。它将积分变为了求和。 一个元球系统定义的标量场为： $$f(x, y, z) = \\sum\_{i=1}^{n} f\_i(r\_i)$$ 其中，$f\_i$ 是第 $i$ 个元球的**势函数 (Potential Function)**，$r\_i$ 是空间点到第 $i$ 个元球中心的距离。

研究者们提出了多种势函数，各有优劣：

*   **Blinn的幂函数**: $f\_i(r) = b\_i e^{-a\_i r^2}$ (高斯函数)，场的影响范围是无限的。
*   **Nishimura的分段二次多项式**: $$f\_i(r) = \\begin{cases} b\_i \\left( 1 - 3(\\frac{r}{R\_i})^2 \\right) & 0 \\le r \\le R\_i/3 \\\\ b\_i \\frac{3}{2} \\left( 1 - \\frac{r}{R\_i} \\right)^2 & R\_i/3 \\le r \\le R\_i \\\\ 0 & r \\ge R\_i \\end{cases}$$ 这个函数在半径 $R\_i$ 之外值为0，具有局部影响性，计算效率更高。
*   **Wyvill的六次多项式**: 提供了更好的融合形状控制。

最终的曲面由方程 $\\sum f\_i(r\_i) = T$ 定义。

### **第五部分：看见隐式曲面——多边形化**

我们有了强大的隐式函数，但还有一个问题：现代的GPU是为渲染三角形而生的。我们如何将一个由数学方程定义的隐式曲面，转换成GPU可以处理的多边形网格呢？这个过程称为**多边形化 (Polygonization)**。

最著名、最经典的算法，就是1987年由Lorensen和Cline提出的**移动立方体 (Marching Cubes)** 算法。这篇论文是计算机图形学史上被引用次数最多的论文之一，至今引用已超过16000次。

**移动立方体算法**

该算法分为两步：

1.  **空间分割**：将我们关心的三维空间区域划分成一个规整的、由许多小立方体（体素，Voxel）组成的网格。
2.  **单元处理**：独立地处理每一个小立方体，找出隐式曲面是如何穿过这个立方体的，并用一或多个三角形来近似这部分曲面。

**核心步骤详解**：

1.  **顶点分类**：对于一个立方体，它有8个顶点。我们计算这8个顶点处的隐式函数值 $f(x,y,z)$，并根据值的符号（正或负）将顶点分类为“在物体内部”或“在物体外部”。
    
2.  **建立索引**：我们将8个顶点的分类结果看作一个8位的二进制数（例如，内部为1，外部为0）。这个数的值域为0-255，可以作为索引。
    
3.  **查询查找表 (Lookup Table)**：总共有 $2^8 = 256$ 种可能的顶点分类组合。但考虑到对称性和互补性，这些组合可以被简化为**15种基本模式**。算法预先构建一个查找表，存储了这256个索引对应的三角剖分方案。根据上一步计算的索引，我们可以直接查表得到，在这个立方体内，应该生成哪些三角形，以及这些三角形的顶点分别位于立方体的哪些边上。
    
4.  **计算顶点位置**：我们已经知道曲面穿过了哪些边，但具体穿在边上的哪个位置呢？这可以通过**线性插值**来计算。假设一条边连接了顶点 $\\mathbf{x}\_1$ 和 $\\mathbf{x}\_2$，它们对应的函数值分别为 $v\_1$ 和 $v\_2$（假设符号相反）。曲面穿过该边的点（函数值为0）的位置 $\\mathbf{x}$ 为： $$\\mathbf{x} = \\frac{|v\_2|}{ |v\_1| + |v\_2| } \\mathbf{x}\_1 + \\frac{|v\_1|}{ |v\_1| + |v\_2| } \\mathbf{x}\_2$$ 这是一个简单的加权平均。
    
5.  **组合三角形**：将所有立方体生成的三角形组合起来，就得到了整个隐式曲面的多边形网格近似。
    

**伪代码**：

```
// Marching Cubes 伪代码
                PolygonMesh MarchingCubes(ScalarField field, float iso_level) {
                    PolygonMesh mesh;
                    for (each cube in field.grid) {
                        // 1. 计算8个顶点的函数值
                        float values[8];
                        for (int i = 0; i < 8; ++i) {
                            values[i] = field.getValue(cube.vertex[i]) - iso_level;
                        }

                        // 2. 建立索引
                        int cube_index = 0;
                        for (int i = 0; i < 8; ++i) {
                            if (values[i] > 0) { // 假设 >0 为内部
                                cube_index |= (1 << i);
                            }
                        }

                        // 如果立方体完全在内部或外部，则没有曲面穿过
                        if (edge_table[cube_index] == 0) continue;

                        // 3. 查询哪些边被穿过，并计算交点
                        glm::vec3 intersection_points[12];
                        for (int i = 0; i < 12; ++i) {
                            if (edge_table[cube_index] & (1 << i)) {
                                // 4. 线性插值计算交点位置
                                intersection_points[i] = linear_interpolate(
                                    cube.edge_start[i], cube.edge_end[i],
                                    values_at_edge_start[i], values_at_edge_end[i]
                                );
                            }
                        }

                        // 5. 根据三角剖分表生成三角形
                        for (int i = 0; tri_table[cube_index][i] != -1; i += 3) {
                            Triangle tri;
                            tri.v1 = intersection_points[tri_table[cube_index][i]];
                            tri.v2 = intersection_points[tri_table[cube_index][i+1]];
                            tri.v3 = intersection_points[tri_table[cube_index][i+2]];
                            mesh.add(tri);
                        }
                    }
                    return mesh;
                }
                
```

算法的精度直接受网格大小的影响。网格越精细，生成的模型就越平滑，但计算量和模型面数也会急剧增加。

### **第六部分：高级课题——自动拟合元球模型**

手动放置成百上千个元球来构建一个复杂模型（如人体）是极其乏味的工作。因此，一个重要的研究方向是：**给定一个已有的几何模型（如多边形网格），如何自动地生成一组元球来逼近它？**

1991年，Muraki 率先提出了一个基于能量最小化的优雅解决方案。

**核心思想**： 将拟合问题转化为一个**最优化问题**。我们定义一个**能量函数 (Energy Function)** $E$，它衡量了当前元球模型与目标模型之间的“差距”。我们的目标就是调整元球的参数，使得这个能量函数 $E$ 达到最小值。

每个元球有5个参数：中心位置 $(x\_i, y\_i, z\_i)$，以及势函数中的两个系数（如Blinn函数中的 $a\_i, b\_i$）。 $$P\_i = (x\_i, y\_i, z\_i, a\_i, b\_i)$$

**能量函数的构成**： 为了得到好的拟合效果，能量函数通常由三部分组成：

1.  **位置能量函数 (Value Energy)**： 首先，我们在目标模型表面上采样 $M$ 个点 $\\Omega = { \\mathbf{p}\_j }\_{j=1}^M$。这一项要求我们生成的元球曲面（等值面）尽可能地穿过这些采样点。即，在采样点 $\\mathbf{p}\_j$ 处，元球系统的函数值 $f(\\mathbf{p}\_j)$ 应该尽可能地接近阈值 $T$。 $$E\_{\\text{value}} = \\sum\_{j=1}^{M} (f(\\mathbf{p}\_j) - T)^2$$
    
2.  **法向能量函数 (Normal Energy)**： 仅仅位置对齐是不够的，我们还希望元球曲面在采样点处的法线方向与目标模型的法线方向一致。设 $\\mathbf{n}\_j$ 是目标模型在 $\\mathbf{p}\_j$ 处的单位法向量，$\\mathbf{N}(\\mathbf{p}\_j)$ 是元球模型在 $\\mathbf{p}\_j$ 处的法向量（即梯度 $\\nabla f(\\mathbf{p}\_j)$）。我们希望这两个向量的夹角尽可能小，也就是它们的点积尽可能接近1。 $$E\_{\\text{normal}} = \\sum\_{j=1}^{M} \\left| \\mathbf{n}\_j - \\frac{\\mathbf{N}(\\mathbf{p}\_j)}{|\\mathbf{N}(\\mathbf{p}\_j)|} \\right|^2$$
    
3.  **收缩能量函数 (Shrink Energy)**： 前两项只在采样点上定义了约束。这可能导致元球为了拟合数据而变得异常“臃肿”或使用奇怪的负权重。收缩能量是一个**正则化项**，它的目的是惩罚那些体积过大的元球，鼓励系统使用更紧凑、更简单的元球组合来解决问题。对于Blinn的势函数，其在整个空间的积分（可以看作元球的“能量”或“体积”）与 $\\frac{|b\_i|}{a\_i^{3/2}}$ 成正比。因此可以定义： $$E\_{\\text{shrink}} = \\sum\_{i=1}^{n} \\left( a\_i^{3/2} |b\_i| \\right)^2$$
    

**复合能量函数**： 最终的总能量是这三项的加权和： $$E = \\frac{1}{M} (E\_{\\text{value}} + \\alpha E\_{\\text{normal}}) + \\beta E\_{\\text{shrink}}$$ 其中 $\\alpha$ 和 $\\beta$ 是权重系数，用于平衡不同约束的重要性。

**求解过程 (迭代分割)**： 直接求解包含 $5n$ 个未知参数的非线性最小化问题非常困难。Muraki提出了一种迭代分割的策略：

1.  **初始化**：从一个元球开始，将其中心置于所有采样点的重心，通过数值优化（如牛顿法）求解其5个参数，使得总能量 $E$ 最小。将这个元球放入一个“元球列表”。
2.  **迭代**：`while (能量 E 仍然过大)` a. **选择**：从元球列表中，根据某种策略（例如，选择分裂后能最大程度降低总能量的那个元球）选出一个元球 $P\_i$ 进行分裂。 b. **分裂**：将 $P\_i$ 从列表中移除，并分裂成两个新的元球 $P'\_i$ 和 $P''\_i$。一个简单的初始分裂策略是：$P'\_i = P''\_i = (x\_i, y\_i, z\_i, a\_i, b\_i/2)$。 c. **优化**：将这两个新的元球加入列表。固定列表中其他元球的参数，只优化 $P'\_i$ 和 $P''\_i$ 的总共10个参数，使得总能量 $E$ 再次达到极小值。
3.  **结束**：重复此过程，直到能量足够小或元球数量达到上限。

### **第七部分：总结与展望**

我们深入探索了隐函数曲面的世界。让我们回顾一下它的优缺点。

**优点**：

*   **易于内外测试**：对碰撞检测等非常友好。
*   **CSG操作简单**：用 `min/max` 即可构建复杂模型。
*   **拓扑变化自然**：完美支持融合、分裂等动画效果。
*   生成的曲面**永远光滑**。
*   **数据量小**：用几百个元球就能表示复杂的有机形态，远少于高精度多边形网格所需的顶点数。

**缺点**：

*   **间接控制**：艺术家无法像编辑多边形网格那样直接拖拽顶点来精确控制曲面，控制不够直观。
*   **难以表示尖锐特征**：由于其内在的平滑性，很难表示模型的棱角、折痕等尖锐细节。（虽然有方法可以缓解，例如使用各向异性的元球，如椭球型元球）。
*   **绘制慢**：直接对隐函数进行光线追踪渲染很慢。虽然可以用Marching Cubes转换为多边形，但这会引入近似误差且生成大量三角面。

尽管存在这些缺点，隐函数曲面因其在特定领域的独特优势，至今仍在图形学中占据着不可替代的地位，尤其是在**流体模拟、有机体建模、医学成像和特效动画**中。
