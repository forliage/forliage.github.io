x86汇编05-3:BCD、逻辑与移位指令

1 十进制调整指令 (Decimal Adjust Instructions)

设计的动机：我们活在十进制世界 计算机内部的一切都是二进制，但人类社会（尤其是金融、商业领域）完全基于十进制。在二进制和十进制之间进行转换是有计算开销的，而且，浮点数表示十进制小数（如0.1）时会存在精度误差。为了解决这些问题，早期的计算机科学家们发明了BCD (Binary-Coded Decimal) 编码。

  • BCD核心思想: 用4个比特来表示一位十进制数（0-9）。
    • 0 -> 0000, 1 -> 0001, ..., 9 -> 1001
    • 1010 (A) 到 1111 (F) 在BCD中是非法的。

两种BCD格式:

  1. 压缩BCD (Packed BCD): 一个字节存储两位十进制数。高4位存一个，低4位存一个。例如，十进制数95表示为字节1001 0101b (95h)。
  2. 非压缩BCD (Unpacked BCD): 一个字节只存储一位十进制数，通常放在低4位，高4位为0。例如，95需要两个字节：0000 1001b (09h) 和 0000 0101b (05h)。

CPU的ALU并不知道什么是BCD，它只会做纯粹的二进制加减法。例如，计算BCD的48 + 35：

  • 二进制表示: 0100 1000b + 0011 0101b = 0111 1101b (7Dh)
  • 7Dh 既不是十进制的 83，也不是BCD的 83h。结果是错误的。

十进制调整指令的唯一目的，就是在二进制加减法之后，对结果进行“修复”，使其变回正确的BCD格式。

1.1 压缩BCD码调整指令

这两条指令都隐式地操作AL寄存器。

1.1.1 daa (Decimal Adjust after Addition)
  • 高级描述: 在执行了ADD或ADC指令后，如果AL中是两个压缩BCD码相加的结果，DAA负责将其调整为正确的压缩BCD码。
  • CPU/架构层面: 这条指令是硬化的逻辑电路，它会检查两个条件来执行修复：
    1. 低4位调整: 如果AL的低4位（低位BCD数）大于9，或者**AF标志位为1**（表示低4位相加时产生了进位），则AL = AL + 6。
    2. 高4位调整: 调整完低4位后，如果AL的高4位（高位BCD数）大于9，或者**CF标志位为1**（表示整个字节相加时产生了进位），则AL = AL + 60h。
  • 系统层面: 这条指令是早期商业应用（如会计软件、收银机系统）的福音。它将一个复杂的、需要多条指令判断和加法的软件逻辑，固化成一条硬件指令，极大地提升了BCD运算的效率。
  • 例子: 计算压缩BCD 48 + 35
    MOV AL, 48h     ; AL = BCD 48
                ADD AL, 35h     ; AL = BCD 35
                ; 执行后: AL = 48h + 35h = 7Dh (0111 1101b)
                ; 此时，标志位AF=1 (因为8+5=13 > 9，低4位有进位), CF=0

                DAA             ; 开始调整
                ; 1. 低4位: 因为AF=1, 所以 AL = 7Dh + 6 = 83h
                ; 2. 高4位: 调整后AL的高4位是8(<=9), 且CF=0, 所以高位不调整
                ; 最终结果: AL = 83h (正确的压缩BCD码)
1.1.2 das (Decimal Adjust after Subtraction)
  • 高级描述: DAA的减法版本，用于SUB或SBB指令之后。
  • CPU/架构层面: 逻辑与DAA类似，但执行的是减法修复：
    1. 低4位调整: 如果AL的低4位大于9，或者**AF=1**（有借位），则AL = AL - 6。
    2. 高4位调整: 调整完低4位后，如果AL的高4位大于9，或者**CF=1**（有借位），则AL = AL - 60h。
  • 例子: 计算压缩BCD 82 - 37
    MOV AL, 82h
                SUB AL, 37h
                ; 执行后: AL = 82h - 37h = 4Bh (0100 1011b)
                ; 此时, AF=1 (2-7需要借位), CF=0

                DAS             ; 开始调整
                ; 1. 低4位: 因为AF=1, 所以 AL = 4Bh - 6 = 45h
                ; 2. 高4位: 调整后高位是4(<=9), CF=0, 不调整
                ; 最终结果: AL = 45h (正确的压缩BCD码)

1.2 非压缩BCD码调整指令

这组指令用于处理每个字节只存一个BCD位的非压缩BCD码。它们通常与ASCII码转换配合使用。

1.2.1 aaa (ASCII Adjust after Addition) & 6.5.2.2 aas (ASCII Adjust after Subtraction)
  • 高级描述: ADD/ADC或SUB/SBB之后，调整AL。它们假定操作数是合法的非压缩BCD码（0-9）。
  • CPU/架构层面:
    • AAA: 如果AL低4位 > 9 或 AF=1，则AL = AL + 6，AH = AH + 1，并设置AF和CF。最后，总是将AL的高4位清零。
    • AAS: 如果AL低4位 > 9 或 AF=1，则AL = AL - 6，AH = AH - 1，并设置AF和CF。最后，总是将AL的高4位清零。
  • 系统层面: 设计用于处理ASCII数字字符串的运算。例如，计算 '7' + '8'： 37h + 38h = 6Fh。运算后，AL的高4位需要被忽略，然后用AAA调整低4位。
1.2.3 aam (ASCII Adjust after Multiplication)
  • 高级描述: 在MUL指令之后，将AL中的二进制结果转换为两位的非压缩BCD码，存放在AH和AL中。
  • CPU/架构层面: 这条指令的本质是：AH = AL / 10 (商)，AL = AL % 10 (余数)。它实际上是一个硬件实现的8位无符号数除以10的指令。虽然名为ASCII调整，但它的通用性使其在需要快速除以10的场合非常有用。
  • 例子: BCD乘法 9 * 8
    MOV AL, 9
                MOV BL, 8
                MUL BL          ; AX = 72 (48h)
                AAM             ; AH = AL / 10 = 72 / 10 = 7
                                ; AL = AL % 10 = 72 % 10 = 2
                                ; 结果 AX = 0702h (非压缩BCD的72)
1.2.4 aad (ASCII Adjust before Division)
  • 高级描述: 在DIV指令之前，将AH和AL中的两位非压缩BCD码转换为一个二进制数存入AL，为除法做准备。
  • CPU/架构层面: 其本质是：AL = AH * 10 + AL。AH随后被清零。这是一个硬件实现的乘10加法。
  • 例子: BCD除法 72 / 9
    MOV AX, 0702h   ; AH=7, AL=2 (非压缩BCD 72)
                AAD             ; AL = AH*10 + AL = 7*10 + 2 = 72 (48h), AH=0
                MOV BL, 9
                DIV BL          ; AL = AL / BL = 72 / 9 = 8 (商)
                                ; AH = 0 (余数)

2 逻辑运算和移位指令

这组指令直接操作数据的二进制位，是底层编程的基石。

2.1 逻辑运算指令

这些指令都会清零CF和OF，并根据结果设置SF, ZF, PF。

2.1.1 and dest, src (Bitwise AND)
  • 高级描述: dest = dest & src。对两个操作数的每一位执行逻辑与。
  • CPU/架构层面: ALU按位执行AND操作。
  • 系统层面 (核心用途): 位屏蔽 (Masking) 或 清零特定位。
  • 例子: 将字符'g' (01100111b) 转换为大写'G' (01000111b)
    MOV AL, 'g'
                AND AL, 11011111b ; or AND AL, 0DFh
                ; 'g'        = 0110 0111
                ; mask       = 1101 1111
                ; --------------------
                ; result (G) = 0100 0111
                ; 该操作将第5位(大小写区别位)强制清零
2.1.2 or dest, src (Bitwise OR)
  • 高级描述: dest = dest | src。按位逻辑或。
  • 系统层面 (核心用途): 设置特定位。
  • 例子: 将字符'G' (01000111b) 转换为小写'g' (01100111b)
    MOV AL, 'G'
                OR AL, 00100000b ; or OR AL, 20h
                ; 'G'        = 0100 0111
                ; mask       = 0010 0000
                ; --------------------
                ; result (g) = 0110 0111
                ; 该操作将第5位强制置1
2.1.3 xor dest, src (Bitwise Exclusive OR)
  • 高级描述: dest = dest ^ src。按位异或。
  • 系统层面 (核心用途):
    1. 翻转特定位: XOR AL, 10000000b 会翻转AL的最高位。
    2. 清零寄存器 (经典优化): XOR EAX, EAX。
      • 设计哲学: 为什么比MOV EAX, 0更好？在很多CPU微架构中，XOR reg, reg指令的机器码更短，且不依赖于立即数单元，执行效率更高，消除了对之前EAX值的数据依赖。
    3. 简单加密/交换数值: A = A^B; B = B^A; A = A^B; 可以不通过第三个变量交换A和B的值。
2.1.4 not op (Bitwise NOT)
  • 高级描述: op = ~op。按位取反（求一的补码）。
  • CPU/架构层面: 这是唯一不影响任何标志位的逻辑运算指令。
2.1.5 test dest, src (Test Bits)
  • 高级描述: CMP的逻辑运算版本。它执行dest & src操作，但只设置标志位，不改变dest的值。
  • CPU/架构层面: ALU执行AND，设置SF, ZF, PF（CF和OF清零），然后丢弃结果。
  • 系统层面 (核心用途): 检查一个或多个位是否为1，而不破坏原始数据。
  • 例子: 检查AX是否为偶数 (检查最低位是否为0)
    MOV AX, 1234h ; 偶数
                TEST AX, 1      ; 测试AX的最低位
                ; 1234h = ...0b, 1 = ...1b. AND结果为0
                ; 标志位ZF=1
                JE is_even      ; 如果ZF=1 (结果为0), 则跳转

                MOV BX, 1235h ; 奇数
                TEST BX, 1
                ; 1235h = ...1b, 1 = ...1b. AND结果为1
                ; 标志位ZF=0
                JNE is_odd      ; 如果ZF=0 (结果非0), 则跳转

2.2 移位和循环移位指令

这些指令将操作数的所有位向左或向右移动指定的次数（count）。count可以是1、CL寄存器或一个8位立即数（80186+）。移出的位通常会进入CF标志位。

逻辑移位
  • 2.2.1 shl dest, count (Shift Logical Left) & 6.6.2.3 sal dest, count (Shift Arithmetic Left)
    • SHL和SAL是完全相同的指令（操作码一样）。它们将位向左移动，空出的最低位用0填充。最高位移入CF。
    • 用途: 快速的无符号/有符号乘法。每左移一位，相当于乘以2。
  • 2.2.2 shr dest, count (Shift Logical Right)
    • 将位向右移动，空出的最高位用0填充。最低位移入CF。
    • 用途: 快速的无符号数除法。每右移一位，相当于除以2。
算术移位
  • 2.2.4 sar dest, count (Shift Arithmetic Right)
    • 将位向右移动，但空出的最高位用原始的符号位填充。
    • 用途: 快速的有符号数除法。SAR能保持负数的符号。
    • 例子对比 SHR 和 SAR:
      MOV AL, 0F0h    ; -16 in signed, 240 in unsigned
                SHR AL, 1       ; AL = 01111000b = 78h (120). 符号变了！

                MOV AL, 0F0h
                SAR AL, 1       ; AL = 11111000b = 0F8h (-8). 符号保持正确！
循环移位
  • 2.2.5 rol dest, count (Rotate Left) & 6.6.2.6 ror dest, count (Rotate Right)
    • 将位循环移动。从一端移出的位会进入另一端，同时也会被复制到CF。数据不会丢失。
    • 用途: 在密码学、图形学中用于数据置换，或者在需要循环移动位域时使用。
带进位的循环移位
  • 2.2.7 rcl dest, count (Rotate through Carry Left) & 6.6.2.8 rcr dest, count (Rotate through Carry Right)
    • 将CF标志位也作为操作数的一部分进行循环。这是一个N+1位的循环（如9位、17位、33位）。
    • 用途 (核心): 多精度移位。
    • 例子: 将一个32位数EDX:EAX逻辑右移一位
      ; 我们想实现 SHR (EDX:EAX), 1
                ; 1. 先移低位EAX，EAX的最低位会进入CF
                SHR EAX, 1
                ; 2. 再带进位移高位EDX。这样，EAX原来的最低位(现在在CF里)
                ;    就会成为EDX的最高位。完美连接！
                RCR EDX, 1
