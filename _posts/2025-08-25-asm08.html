x86汇编08:中断程序设计

引言：为什么需要中断？

想象一下，你要写一个程序，等待用户按下一个键。你该怎么做？

  • 愚蠢的方法：轮询 (Polling)

    wait_for_key:
                    ; 检查键盘状态端口，看是否有按键
                    IN AL, KEYBOARD_STATUS_PORT
                    TEST AL, KEY_PRESSED_BIT
                    JZ wait_for_key  ; 如果没有按键，就跳回去继续检查
                ; ... 按键被按下，继续处理 ...

    这种方法的致命缺陷是：在用户没有按键的漫长时间里，CPU 100% 都在执行这个无意义的循环，浪费了数以亿计的时钟周期。这就像一个焦急的门卫，每秒钟都要把门打开看看有没有人来，而不是在有人敲门时再去开门。

  • 智慧的方法：中断 (Interrupts) CPU正常执行其他任务。当用户按下键盘时，键盘控制器会向CPU发送一个电信号。CPU会立即暂停当前正在执行的任务，自动跳转到一段专门处理键盘输入的代码（即中断服务程序），处理完毕后，再无缝地恢复到之前被暂停的任务。

设计的核心哲学: 中断机制将CPU从**“主动、持续的监视”中解放出来，使其可以专注于计算任务，只在有“事件”发生时才被动地响应。这是从“过程驱动”到“事件驱动”**编程模型的根本性转变，是所有现代操作系统和图形用户界面的基石。

中断的分类:

  1. 外部中断 (Hardware Interrupts): 来自CPU外部的硬件设备（键盘、鼠标、时钟、网卡）。
  2. 内部中断 (Software Interrupts & Exceptions):
    • 软件中断: 由INT指令主动触发，是应用程序请求操作系统服务的规范方式。
    • 异常 (Exceptions): 由CPU在执行指令时检测到的错误或特殊情况（如除以零、访问无效内存）所引发。

1 中断处理的CPU级过程”

当中断信号到达CPU时，CPU内部会发生一系列由硬件自动完成的、不可打断的精确操作：

  1. 完成当前指令: CPU会执行完当前正在执行的指令，以保证指令的原子性。
  2. 保存断点 (Context Switching - Part 1): 这是中断机制的精髓。CPU会自动将三个关键的寄存器压入堆栈：
    • 标志寄存器 (EFLAGS)
    • 代码段寄存器 (CS)
    • 指令指针 (EIP) 这个“断点”信息，是CPU能够完美回到被打断之处的唯一凭证。
  3. 查找中断处理程序地址:
    • 在内存的最低地址处（实模式下从0000:0000开始），有一个256项的中断向量表 (Interrupt Vector Table, IVT)。
    • 每个表项是一个4字节的远指针（CS:IP）。
    • 中断号（0-255）被用作这个表的索引。例如，对于INT 8h，CPU会查找第8个表项。
  4. 跳转到处理程序: CPU从IVT中取出CS:IP地址，加载到自己的CS和IP寄存器中，从而跳转到中断服务程序（ISR）的入口点。
  5. （对于硬件中断）清除中断请求: CPU通常会自动清除内部的中断请求标志，并可能禁止响应同级或低级的中断（通过IF标志位），以防止中断风暴。

2 中断服务程序（ISR）的设计准则

一个合格的ISR必须像一个“优雅的客人”：悄悄地来，做完事，不留下一丝痕迹地离开。

  1. 保护现场: CPU只自动保存了EFLAGS, CS, EIP。ISR必须在开头手动保存所有它将要用到的通用寄存器（PUSHAD或逐个PUSH），并在结尾按相反顺序恢复（POPAD或逐个POP）。
  2. 执行任务: 这是ISR的核心逻辑，应该尽可能地快。ISR执行期间，整个系统（或至少同级中断）可能是被阻塞的。
  3. （对于硬件中断）发送EOI: 必须向可编程中断控制器 (PIC, 8259A) 发送一个中断结束 (End Of Interrupt) 信号（通常是向端口20h写入20h），告知它可以继续发送下一个中断了。
  4. 返回: 必须使用IRET (Interrupt Return) 指令返回。IRET会自动恢复EIP, CS, EFLAGS，让被打断的程序恢复运行。

3 时钟中断程序设计 (Timer Interrupt)

  • 背景: PC中有一个可编程间隔定时器（PIT, 8253/4），它以约18.2次/秒（1193180Hz / 65536）的频率产生IRQ0硬件中断，对应**INT 8h**。

  • 系统用途: DOS利用这个中断来更新系统时间、日期计数器。

  • 我们的目标: 编写一个程序，在屏幕的固定位置显示一个每秒变化的“时钟”。

  • 设计模式：中断挂钩 (Hooking an Interrupt) 我们不能粗暴地直接用我们的ISR地址覆盖INT 8h的向量，因为那样系统时间就不走了。正确的做法是：

    1. 用DOS功能调用(INT 21h, AH=35h)获取并保存原始INT 8h处理程序的地址。
    2. 用DOS功能调用(INT 21h, AH=25h)将INT 8h的向量设置为我们自己的ISR地址。
    3. 在我们的ISR的末尾，通过一个远跳转JMP来调用原始的ISR。这称为中断链 (Chaining)。

  • 例子:

    ; --- 数据段 ---
                old_timer_vector DD ?    ; 保存原始INT 8h向量
                tick_count       DB 0    ; 我们的时钟计数器

                ; --- 我们的时钟中断ISR ---
                timer_isr PROC FAR
                    ; 1. 保护现场
                    PUSHAD
                    PUSH DS
                    MOV AX, @DATA
                    MOV DS, AX

                    ; 2. 执行任务
                    INC [tick_count]
                    CMP [tick_count], 18 ; 大约1秒 (18.2 ticks)
                    JNE skip_display
                    
                    MOV [tick_count], 0  ; 重置计数器
                    ; ... 此处添加在屏幕上显示时间的逻辑 ...
                    
                skip_display:
                    ; 3. 发送EOI (可选，因为我们会链式调用)
                    ; MOV AL, 20h
                    ; OUT 20h, AL

                    ; 4. 恢复现场 (除了要跳转的寄存器)
                    POP DS
                    POPAD

                    ; 5. 链式调用原始ISR
                    JMP [old_timer_vector]
                timer_isr ENDP

                ; --- 主程序 ---
                main PROC
                    ; --- 安装ISR ---
                    MOV AX, 3508h       ; AH=35h, AL=08h (获取INT 8h向量)
                    INT 21h             ; 返回在 ES:BX
                    MOV WORD PTR [old_timer_vector], BX
                    MOV WORD PTR [old_timer_vector+2], ES

                    PUSH DS
                    MOV DX, OFFSET timer_isr
                    MOV AX, SEG timer_isr
                    MOV DS, AX
                    MOV AX, 2508h       ; AH=25h, AL=08h (设置INT 8h向量)
                    INT 21h
                    POP DS
                    
                    ; ... 主程序可以做其他事, 或进入一个等待退出的循环 ...
                main ENDP

4 键盘中断程序设计 (Keyboard Interrupt)

  • 背景: 键盘控制器在有按键动作时，产生IRQ1硬件中断，对应**INT 9h**。

  • 系统用途: 原始的INT 9h ISR会从端口60h读取按键的扫描码 (Scan Code)，将其转换为ASCII码，然后放入键盘缓冲区。

  • 我们的目标: 编写一个程序，当用户按下F1键时，屏幕背景变色。

  • 设计模式: 与时钟中断完全相同，通过挂钩和链式调用来实现。

  • 例子:

    ; --- 我们的键盘ISR ---
                keyboard_isr PROC FAR
                    ; 1. 保护现场 (至少需要保存AX)
                    PUSH AX

                    ; 2. 执行任务
                    IN AL, 60h          ; 从端口60h读取扫描码
                    CMP AL, 3Bh         ; 3Bh是F1键的扫描码
                    JNE chain_to_old    ; 如果不是F1键，直接跳到链式调用
                    
                    ; ... 此处添加改变屏幕背景色的逻辑 ...

                chain_to_old:
                    POP AX
                    ; 3. 链式调用
                    JMP [old_keyboard_vector]
                keyboard_isr ENDP

                ; --- 主程序安装部分与时钟中断类似, 只是中断号是09h ---

5 单步中断程序设计 (Single-Step Interrupt)

  • 背景: 这是由CPU内部产生的异常，中断号为1。它不是由硬件设备触发的，而是由EFLAGS寄存器中的陷阱标志 (Trap Flag, TF) 控制的。

  • CPU架构层面: 当TF位被置为1时，CPU在成功执行完一条指令后，在执行下一条指令前，会自动产生一个INT 1异常。

  • 系统用途: 这是所有调试器 (Debugger) 实现单步执行 (F7或Step Into) 功能的硬件基础。

  • 设计模式:

    1. 调试器程序首先挂钩INT 1。
    2. 当需要单步执行被调试程序时，调试器会修改被调试程序在堆栈上的EFLAGS镜像（在CALL或INT后），将其TF位置为1。
    3. 当被调试程序通过IRET或RETF返回时，TF=1的EFLAGS被加载，单步模式启动。
    4. 被调试程序执行一条指令后，INT 1触发，控制权回到调试器的ISR。
    5. 调试器的ISR可以显示寄存器状态，然后再次设置TF位，让被调试程序继续执行下一条指令。

  • 例子: 一个简单的程序，每执行一条指令就打印一条消息

    ; --- 我们的单步ISR ---
                single_step_isr PROC
                    ; 保护现场
                    ; ... 打印 "One step executed!" 的逻辑 ...
                    ; 恢复现场
                    IRET  ; 直接IRET，因为TF在EFLAGS中，返回后会自动再次触发
                single_step_isr ENDP

                ; --- 主程序 ---
                ; 1. 挂钩INT 1
                ; ... (代码类似上面)

                ; 2. 设置TF标志位
                PUSHF               ; 将当前FLAGS压栈
                POP AX              ; 放入AX
                OR AX, 0100h        ; 将TF位置1 (bit 8)
                PUSH AX             ; 压回
                POPF                ; 恢复到FLAGS，单步模式开启！

                ; --- 被调试的代码段 ---
                MOV CX, 10          ; 执行完这条，INT 1触发
                XOR BX, BX          ; 执行完这条，INT 1触发
                ...

6 驻留程序设计 (TSR - Terminate and Stay Resident)

  • 背景: 这是一个DOS时代的经典编程模型，但其思想影响深远。

  • 目标: 如何让一个程序（特别是它的ISR）在“终止”后，依然“存活”在内存中，继续响应中断？

  • 设计模式:

    1. 程序正常加载到内存。
    2. 程序执行其初始化代码：挂钩它需要的中断（如时钟、键盘）。
    3. 程序计算出它的ISR和相关数据总共占用了多少内存。
    4. 程序不使用常规的INT 21h, AH=4Ch终止，而是调用一个特殊的DOS功能：INT 21h, AH=31h (Keep Process)。
    5. 这个功能调用会告诉DOS：“请把控制权还给你，但不要回收我指定的这部分内存”。
    6. DOS返回命令提示符，但我们的ISR代码块依然完好地留在内存中，像一个“幽灵”一样，每当时钟滴答或键盘按下时，它就会被硬件唤醒。

  • 系统层面: 这是一种原始的“后台服务”或“守护进程”的实现方式。输入法、病毒、内存管理工具等在DOS时代都广泛使用TSR技术。它展示了程序如何能与操作系统进行“协商”，修改系统的默认行为，并成为系统的一部分。

  • 例子: 一个简单的TSR，当用户在任何程序中按下Ctrl+C时，除了执行默认的终止操作外，还让PC喇叭响一声。

    ; --- 程序的启动部分 ---
                start:
                    ; 1. 挂钩键盘中断 INT 9h
                    ; ... (代码略)

                    ; 2. 计算需要驻留的内存大小
                    ; 假设我们的ISR代码和数据从_start到_end
                    MOV DX, (OFFSET _end - OFFSET start + 15) / 16 + 1
                    
                    ; 3. 调用TSR终止功能
                    MOV AX, 3100h       ; AH=31h (Keep), AL=00h (返回码)
                    INT 21h

                ; --- 驻留部分 (这部分代码会留在内存中) ---
                keyboard_isr PROC FAR
                    ; ...
                    ; 检查是否是Ctrl+C组合键
                    ; 如果是，执行发声逻辑
                    ; ...
                    ; 链式调用原始INT 9h
                    JMP [old_keyboard_vector]
                keyboard_isr ENDP
                ; ... 其他数据和变量 ...
                _end:
